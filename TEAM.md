# LAXTHA 웹 리뉴얼 팀 (AI Orchestra)

> "각 AI/모델이 가장 잘하는 일을 맡는다." — 하나의 결과물을 위한 다중 모델 협업 체계.

## 팀 구성 & 역할 배정

| 역할 | 담당 | 책임 | 이유 |
|---|---|---|---|
| **PM / 아키텍트 / 리뷰어** | Claude Opus 4.8 | 디자인 시스템 해석, 정보구조 설계, 빌드 스펙 작성, 코드 리뷰, 최종 품질 관리 | 긴 맥락 유지·설계 판단·일관성 검수에 강함 |
| **시니어 프론트엔드 엔지니어** | Codex CLI · GPT-5.5 | 핵심 구현(HTML/CSS/JS), 컴포넌트 조립, 인터랙션 | 정밀 구현·코드 품질 |
| **구현 보조 / 데이터·반복작업** | Codex CLI · GPT-5.5 | 제품 데이터 정리, 반복 마크업, 빌드/프리뷰 실행, 테스트 | 빠른 반복·대량 작업 |

> 참고: `gpt-5.6`은 현재 ChatGPT 계정 인증 Codex에서 미지원(400 에러) → 구현은 **gpt-5.5(high reasoning)** 로 수행. API 키 인증으로 전환하면 5.6 사용 가능하나 토큰당 과금 발생하므로 사용자 확인 전까지 5.5 사용.
| **비주얼 자산** | (보류) | 제품 사진은 **원본 유지** (아래 정책 참조) | 제품 식별성 보존 + 이미지 생성 과금 회피 |

## 운영 원칙 (전역 규칙 준수)
1. **Codex 위임**: 코드 구현·파일 생성·반복 실행·테스트는 Codex가 수행. Claude는 설계·검토·방향·최종 확인.
2. **품질 우선**: 토큰 절약이 품질 타협의 이유가 되지 않음. 중요한 구현은 GPT-5.6, 단순 반복은 GPT-5.5.
3. **결제/과금**: Codex는 ChatGPT 구독 인증(auth.json) 기반 → 토큰당 추가 과금 없음. 이미지 생성 등 별도 과금이 필요한 작업은 사용자 사전 확인.

## 제품 사진 정책 (중요)
- 제품 사진은 **AI로 재생성하지 않는다.** 재생성 시 ①제품이 실제와 달라져 식별 불가 위험, ②이미지 생성 API 과금 발생.
- 따라서 원본 제품 이미지를 `assets/products/`에 **1:1 슬롯**으로 배치하고, 디자인은 레이아웃·타이포·프레젠테이션만 업그레이드한다.
- 장식용 비주얼(히어로 배경 등)은 사진 대신 **CSS/SVG 생체신호 파형 모션**으로 처리 → 무과금·고품질.

## 호출 방법 (Codex CLI)
```bash
# 고품질 핵심 구현
codex exec -m gpt-5.6 -s workspace-write -C <projdir> "<spec>"
# 반복/데이터 작업
codex exec -m gpt-5.5 -s workspace-write -C <projdir> "<spec>"
```
